# 2026-05-25 学习日志

## 知识点

- [AI / Skill] 在新机器上部署 CodeBuddy Skill 的标准流程（克隆源码 + 复制到 `~/.codebuddy/skills/`） → 详见 [topics/ai/skill/](/topics/ai/skill/)
- [AI / Skill] SKILL.md 中的"本地路径"必须用 `<workspace>/xxx` 占位符，禁止硬编码绝对路径 → 详见 [topics/ai/skill/](/topics/ai/skill/)

## 问题与解决

- **问题**: 在新机器（工作区 `/data/workspace`）上克隆并安装 `skill-knowledge-writer` 后，发现 SKILL.md 里写的本地路径是 `~/.openclaw/workspace/knowledge-base`，与当前实际路径不一致，skill 触发时会找错位置。
- **解决**:
  1. 把源仓库克隆到工作区作为可维护的源码：`git clone https://github.com/SayGoodBey/skill-knowledge-writer.git /data/workspace/skill-knowledge-writer`
  2. 复制到 CodeBuddy 全局 skill 目录使其生效：`cp -r /data/workspace/skill-knowledge-writer ~/.codebuddy/skills/knowledge-writer`（注意目录名必须等于 SKILL.md frontmatter 的 `name` 字段）
  3. 同步修改两份 SKILL.md（源仓库 + 已安装版），把硬编码路径改为通用占位 `<workspace>/knowledge-base`，附带说明"由 skill 在运行时自适应解析"

## 今日总结

把这台新机器配成了知识沉淀工作站：

1. **knowledge-base 仓库**：直接克隆到 `/data/workspace/knowledge-base`，作为 VitePress 文档源
2. **knowledge-writer skill**：源码维持在 `/data/workspace/skill-knowledge-writer`（保留 `.git` 便于后续推送），同时复制到 `~/.codebuddy/skills/knowledge-writer` 让 CodeBuddy 全局识别
3. **填掉一个坑**：SKILL.md 中所有"本地路径"必须用占位符，**多机器场景下绝对路径就是定时炸弹**

教训：跨机器维护的 skill / 配置文件，凡是涉及路径的字段都应优先使用变量或占位符，写死等于在新环境提前埋雷。今后新建 skill 时把这一条加进自查清单。
