## 学习总结

今天用集装箱-轮船的比喻完整学习了 Kubernetes 的 11 个核心概念，形成了从底层到顶层的完整知识链：

```
kubectl → API Server → etcd → Scheduler → kubelet → Pod（拉容器）
                                 ↓
                          Controller（持续调和）
                                 ↓
                   Deployment → ReplicaSet → Pod × N
                                 ↓
                        Service（固定虚拟 IP）
                         ↑ Label Selector
                        Pod ← Volume ← ConfigMap/Secret/PVC
                       Namespace 隔离一切
```

### 11 个核心概念速查

| 概念 | 一句话 |
|------|--------|
| API Server | 集群唯一入口，所有操作必经 |
| etcd | 集群内存，存所有对象的真实状态 |
| Controller | 持续对比期望 vs 实际，趋近期望 |
| Scheduler | 为 Pod 选节点（过滤 + 打分） |
| kubelet | 节点代理，watch + 启容器 |
| Pod | 最小调度单元，共享网络和存储 |
| Deployment | 管理 Pod 版本和副本数 |
| ReplicaSet | 维持指定数量的 Pod 副本 |
| Service | 稳定虚拟 IP，Pod 随便换外面无感知 |
| Namespace | 逻辑租户隔离 |
| Label/Selector | 松耦合关联机制 |

### 核心设计思想

1. **声明式 API**：描述目标状态，不描述操作步骤
2. **松耦合**：所有组件只与 API Server 通信，通过 watch 事件异步协作
3. **Label/Selector**：资源关联靠标签匹配，不靠硬编码
4. **调和循环**：Controller 持续驱动实际状态趋近期望状态
