# 2026-05-21 学习日志

## 知识点

- [前端 / Next.js] Next.js 16 build 阶段 RangeError：barrel re-export 触发模块加载链爆栈 → 详见 [topics/frontend/next/](/topics/frontend/next/)
- [项目 / DRMS] Next 16 升级后 `next build` 在 page-data 收集阶段报 `RangeError: Maximum call stack size exceeded at ignore-listed frames`，含完整二分定位过程 → 详见 [topics/projects/drms/](/topics/projects/drms/)

## 问题与解决

- **问题**: DRMS 升级 Next 15 → 16 后 `npm run build` 在 "Collecting page data" 阶段崩溃，错误栈被 Next 标记为 `ignore-listed frames` 完全隐藏，看不到任何业务调用栈。
- **解决**:
  1. **手工二分定位法**：把所有 `pages/*` 替换成最小骨架确认 build 能通过 → 逐一恢复 → 锁定 `_document.tsx` → 在它内部二分 imports → 命中 `@config + @lib` 同时引就炸 → 沿 `@lib` barrel 向下挖到 `lib/csrf/middleware/csrf.ts` 顶层 `import { logger } from '../../log4js'` 是触发点
  2. **修复（4 处）**：
     - `lib/index.ts` 注释 `export * from './grpc/client'` 和 `export * from './log4js'`（barrel 不再转发副作用模块）
     - `lib/csrf/middleware/csrf.ts` 把顶层 `import logger` 改成函数内 lazy require（**核心修复**，验证过：改回顶层立即重现 RangeError）
     - `lib/grpc/client.ts` 同上
     - `pages/[[...slug]].tsx` 同上
     - 所有原本 `from '@lib'` 取 `logger / taskClient` 的代码切到子路径直接 import（`@lib/log4js` / `@lib/grpc/client`）
  3. **结果**：完整真实代码下 `npm run build` 通过（编译 ~5s，所有路由生成成功），`next.config.js` 不需要任何 experimental hack。

## 走过的弯路（记录避免下次再踩）

- ❌ 误判 Node 版本（猜是 18，实际 20.20.2 一样炸）
- ❌ 在 `next.config.js` 顶层注册 `process.on('unhandledRejection')` 抢栈打印 → 失效，Next 内部 handler 已先处理
- ❌ `experimental.cpus: 1` + `workerThreads: false` 单进程化 → 仍然炸（说明跟 worker 无关，是模块加载问题）
- ❌ `webpack.optimization` 关闭 `usedExports/sideEffects/concatenateModules` → 无效（树摇优化层面解决不了运行时 require 递归）

## 今日总结

`ignore-listed frames` 类问题是 Next 16 的"高难度调试场景"——常规手段（栈、debugger、process hook、webpack 调参）全部失效。**唯一稳定的定位方法是手工二分页面 + 二分 imports + 二分 barrel 链路**。

修复手法本质是两个：
1. **barrel 不要转发带顶层副作用的子模块**（log4js.configure、grpc proto 加载、socket.io 初始化等）
2. **被 page-data worker 间接加载的中间模块，对副作用模块改用 lazy require**（彻底切断顶层加载链，比 `sideEffects: false` 等优化提示更彻底）

跨文件风格不必强求统一：`lib/` 内会被 page-data 加载的模块用 lazy require，`pages/api/*` 用顶层 import，这种"按位置分类"的不一致是合理的。

教训层面：升级 Next 大版本时，**barrel 互引环 + 顶层副作用 + commonjs 互操作**是高风险三件套；以后类似升级前先跑 `madge --circular` 暴露循环，比事后 debug 效率高数十倍。
